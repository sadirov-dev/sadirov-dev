# Repository Branding Guide

## Recommended Pinned Set

1. `edutc-platform`
2. `ai-system-lab`
3. `react-ui-lab`
4. `fastapi-core`
5. `docker-stack-starter`
6. `sadirov-dev`

## Naming Principles

- Prefer short, intentional names over generic tutorial naming
- Avoid names like `final-project`, `new-project`, `react-app`, or `test-api`
- Use names that communicate product value or technical role

## Description Formula

`What it is` + `what stack it uses` + `why it matters`

Example:

`Production-oriented FastAPI backend for authentication, PostgreSQL workflows, and clean service architecture.`

## Topic Strategy

- Use 4 to 6 high-signal topics
- Keep them consistent with the repository’s real role
- Good examples: `react`, `fastapi`, `postgresql`, `docker`, `ai`, `fullstack`, `tailwindcss`

## README Formula For Strong Repositories

1. One-line project summary
2. Screenshot or UI preview if visual
3. Tech stack
4. Key features
5. Architecture or folder structure
6. Setup instructions
7. Deployment or demo
8. Roadmap or lessons learned

## What To Hide Or Improve

- Hide half-finished tutorials with weak naming
- Improve repositories with missing screenshots or empty README files
- Remove cluttered badges and low-value widgets
- Keep public work aligned with the `fullstack + AI systems + premium engineering` narrative
